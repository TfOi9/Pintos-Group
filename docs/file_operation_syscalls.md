# File Operation Syscalls

在这一任务中，我们需要实现文件系统调用。对每一个进程，我们需要维护一个文件描述符列表，并支持文件打开、关闭和读写。当前的文件系统暂不支持并发操作，因此我们需要一把总的文件系统锁。此外，我们还需要更新 `fork` 的实现，支持文件描述符继承。

### 数据结构设计

我们采用 `fd_entry` 来存储文件描述符，它包括以下字段：

- `fd`：文件描述符编号；
- `handle`：文件句柄（见下文）；
- `elem`：链表项

`fd_entry` 以链表结点的形式存储在 PCB 的 `fd_list` 链表下。
此外，我们在 PCB 中新引入了 `next_fd` 编号，表示下一个可用的文件描述符编号。

对于文件句柄，由于其需要实现文件的自动关闭，需要维护引用计数。文件句柄包括以下字段：

- `file`：文件指针；
- `ref_cnt`：引用计数；
- `lock`：互斥锁，用于保证文件句柄线程安全

我们使用函数 `file_handle_create` 创建文件句柄。共享地获取文件句柄所有权时，需要显式调用 `file_handle_get` 来增加引用计数。不再需要文件句柄时，需要调用 `file_handle_put` 释放所有权，减少引用计数。计数归零时，文件句柄会自动关闭文件并自我销毁。

### 工具函数

- `fd_lookup` 用于在指定进程 PCB 下查找文件描述符对应的 `fd_entry`。
- `fd_install` 用于将一个文件指针插入到指定进程 PCB 中，返回新的文件描述符编号。
- `fd_close` 用于将指定文件描述符关闭，删除其在 PCB 中的 `fd_entry`。
- `fd_close_all` 用于关闭一个进程所有文件描述符，并清理所有相关数据结构。常用于进程退出时。

### 系统调用概述

对于 `create` `remove` `open` `close` `filesize` `tell` `seek`等调用，只需要上锁、按要求操作文件、更新 PCB 状态、解锁即可。

### `read` 系统调用

`fd` 为 0 时表示 `STDIN`，直接从标准输入流读入；`fd` 大于等于 2 时从相关文件描述符读入。为了保证内存不溢出，采取分块读法。

### `write` 系统调用

`fd` 为 1 时表示 `STDOUT`，直接从标准输入流读入；`fd` 大于等于 2 时从相关文件描述符输出。采取分块写法。

### 可执行文件保护

为了防止可执行文件运行时被修改，我们在 PCB 中存储可执行文件的指针，并在开始运行前禁止修改，在运行结束后允许修改即可。

### `fork` 的更新

需要新增文件描述符复制。我们创建新的 `fd_entry` 及其链表，但不复制文件句柄，而是直接浅拷贝并增加引用计数即可。
