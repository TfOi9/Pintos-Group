## Process Control Syscalls

在先前的默认 syscall 处理代码中，直接对 `args[idx]` 解引用。如果此时实际传入的参数错误，将导致（内核）非法内存访问，直接炸内核。因此，我们需要先实现用户内存访问合法性判断，以及安全拷贝用户内存内容。

### 用户内存访问合法性判断

用户内存访问的合法性可以按如下流程判定：

- 用户内存不为 NULL；
- 用户内存地址未出界（使用 `validate_user_uaddr` 函数判断目标内存是否属于用户内存区）；
- 当前线程有合法的进程控制块，且控制块持有合法的页指针；
- 在页指针对应的页中，能够找到目标内存。

函数 `validate_user_uaddr` 实现了这一功能。

### 安全拷贝用户内存

拷贝内存只需要逐个判断各字节的访问合法性，并复制到目标缓冲区即可。

- `read_user_u8` 实现了安全访问一个字节并写入缓冲区；
- `copy_in` 实现了类似于 `memcpy` 的用户内存到内核缓冲区的内存复制；
- `fetch_arg_u32` 实现了安全读取一个 32 位无符号整数函数参数；
- `fetch_arg_u32` 实现了安全读取一系列 32 位无符号整数函数参数。

上述函数均返回 `bool` 类型，表示读取是否成功。若不成功，写入缓冲区中值的将会是无效值。
